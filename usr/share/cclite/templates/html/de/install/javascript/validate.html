This form validation tool can check for erforderlichen, Integer, e-Mail und Datum-Felder in your forms. It is easily portable to all your forms and will help save server side resources from form field validation and page reloads. All the messages are customized in the hidden form element where you also select the validation requirement. You can mix the required element with any of the other elements. This code is completely portable and can be used with any form by just adding your hidden elements appropriately and adding the onClick command to a button.

Die Regeln.
1) An set a required field use your existing field name and add an r_ to the beginning of the field name and create a hidden element.
BEISPIEL:
<input type="text" name="your_fieldname">
<input type="hidden" name="r_your_fieldname" value="Required error message goes here">

2) An set an integer field use your existing field name and add an i_ to the beginning of the field name and create a hidden element.
BEISPIEL:
<input type="text" name="your_fieldname">
<input type="hidden" name="i_your_fieldname" value="Integer error message goes here">

3) An set a date field use your existing field name and add an d_ to the beginning of the field name and create a hidden element. This date field only accepts the format MM/DD/YY or M/D/YY.
BEISPIEL:
<input type="text" name="your_fieldname">
<input type="hidden" name="d_your_fieldname" value="Datum error message goes here">

4) An validate an email field use your existing field name and add an e_ to the beginning of the field name and create a hidden element.
BEISPIEL:
<input type="text" name="your_fieldname">
<input type="hidden" name="e_your_fieldname" value="The email address format is incorrect">

Die Initialisierung des Skripts.
You MUST use the type="button" NOT THE type="submit" for your SUBMIT button, the javascript sends the submit for you.
BEISPIEL:
<input type=button Wert="Button Message Here" onclick="check(form,form.elements.length)">
