All the messages are customized in the hidden form element where you also select the validation requirement. You can mix the required element with any of the other elements. This code is completely portable and can be used with any form by just adding your hidden elements appropriately and adding the onClick command to a button.

Seguindo as regras.
1) Para set a required field use your existing field name and add an r_ to the beginning of the field name and create a hidden element.
EXEMPLO:
<input type="text" name="your_fieldname">
<input type="hidden" name="r_your_fieldname" value="Required error message goes here">

2) Para set an integer field use your existing field name and add an i_ to the beginning of the field name and create a hidden element.
EXEMPLO:
<input type="text" name="your_fieldname">
<input type="hidden" name="i_your_fieldname" value="Integer error message goes here">

3) Para set a date field use your existing field name and add an d_ to the beginning of the field name and create a hidden element. This date field only accepts the format MM/DD/YY or M/D/YY.
EXEMPLO:
<input type="text" name="your_fieldname">
<input type="hidden" name="d_your_fieldname" value="Data error message goes here">

4) Para validate an email field use your existing field name and add an e_ to the beginning of the field name and create a hidden element.
EXEMPLO:
<input type="text" name="your_fieldname">
<input type="hidden" name="e_your_fieldname" value="The email address format is incorrect">
